# Контекст проекта «Дашборд образования СПб»

 **Проект носит информационных характер**

> Данные вымышленные
---

## 1. Кто и зачем

- **Заказчик:** ГКУ «Центр архивных документов» СПб (ЦАД), сайт `portal.gkuoa.ru`
- **Задача:** интерактивный аналитический дашборд по образованию СПб, который потом будет встроен на сайт компании
- **Аудитория:** комитеты, журналисты, родители, исследователи


## 3. Стек дашборда

- **Tabler** (Bootstrap-based) — UI-каркас, сетка, карточки, табы. CDN, MIT
- **Apache ECharts** — все графики. CDN, MIT
- **Choices.js** — красивые dropdown-ы с поиском
- **CountUp.js** — анимация KPI-цифр
- **Tabler Icons** — иконки
- **Inter** (Google Fonts) — шрифт

Всё через CDN, никаких сборок (Vite/Webpack нет). Локальный запуск через **Live Server** в VS Code (CORS-политика блокирует `file://` для fetch).

---

## 4. Структура `index.html` (логические блоки)

1. **Шапка** — лого-флюгер Адмиралтейства (SVG) + название + переключатель темы
2. **Декоративный силуэт города** под шапкой
3. **Sticky панель фильтров:** тип (чипы ДОУ/СОШ/СПО), район (multiselect), форма собственности (multiselect), год (select), поиск по учреждению
4. **4 KPI-плашки** с иконками
5. **5 блока графиков 2×2:**
   - **БЛОК А** (Сеть учреждений): donut по типам · hbar по районам · donut по формам
   - **БЛОК Б** (Контингент): donut сады (3 возраста) · funnel школы (3 ступени) · vertical bar параллели 1–11
   - **БЛОК В** (Вакансии): treemap по районам (плавный градиент) · hbar ТОП-15 должностей (градиент)
   - **БЛОК Г** (Кросс-аналитика, наполняемость)
   - **БЛОК Д** (Население)
7. **Футер** с источниками
